Intro and header

For this data story template we wanted a simplified page. Accordingly we removed elements of the header, preserving only the main black header bar, the link back to data stories, and the translation function. These are responsive but with less transformation across screen sizes.

The title itself uses class titleheader to get that transparent overlay that ensures readability on a wide variety of title image slides.

Story anchors

Right now we have the headers IDed as the sections. Datastories.css includes storyanchor class to help align things below the sticky header. For header anchors, use class storyanchor. This chapter anchor functionality can be easily commented out for simple data stories. They also collapse into a Contents expander on mobile.

This is set so that on page load, the "Intro" nav button is active - highlighted. This plus scrolling should give the user a clear understanding of what that sticky anchor-nav does.

For complicated chaptered stories we can also set different background colors to differentiate where we are in the story. Currently, the image hover expansion is commented out of datastories.css to make development easier.

As with previous data stories, the title image is set by title1.jpg in the story's folder.

Widths

This uses a div class .narrow for the main sections. This sets its width. In order to have wider sections (for images and data visualization) or full-width sections (for superwide data vis), you'll close the narrow div and open a new medium or wide one using class .medium or .wide. You can see a medium one below. These classes come from the Core Framework Documentation.
